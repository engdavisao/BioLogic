## 📜 Descrição

**BioLogic** é um jogo educacional digital acessível, desenvolvido em parceria com a UNIFESP, voltado para estudantes do 1º ano do Ensino Médio com deficiência visual que utilizam eye-tracker como tecnologia assistiva. O jogo roda diretamente no navegador, sem necessidade de instalação, e foi construído com a biblioteca Phaser 3.

O problema central que o BioLogic endereça é a escassez de materiais digitais adequados para esse público: a maioria dos conteúdos educacionais exige leitura extensa, gera fadiga visual e não oferece compatibilidade com dispositivos de rastreamento ocular. O resultado costuma ser um processo de aprendizagem ineficiente que compromete a autonomia e o engajamento do aluno.

Para resolver isso, o jogo adota uma mecânica de quiz com 15 questões de múltipla escolha que cobrem os principais temas de Biologia do 1º ano: citologia, mitose, meiose, evolução e genética. O fluxo é linear e previsível, o jogador avança pelo menu inicial, passa pelo tutorial e responde as questões sequencialmente. Ao errar, recebe feedback explicativo e permanece na questão até acertar; ao acertar, recebe uma explicação do conteúdo antes de avançar. Ao final, é exibido um resumo geral de desempenho.

A interface foi projetada segundo princípios de design universal e acessibilidade visual: alto contraste, tipografia ampliada, layout minimalista sem poluição visual e navegação simples o suficiente para ser operada integralmente pelo olhar. O áudio é restrito a efeitos sonoros curtos de acerto e erro, a narração da introdução e fechamentos explicativos, evitando sobrecarga sensorial. O mouse funciona como alternativa equivalente ao eye-tracker em qualquer contexto.

Um diferencial da mecânica são as "pegadinhas": em algumas questões, a resposta correta pode estar fora das alternativas convencionais: em imagens, no enunciado ou em outros elementos da tela, exigindo atenção e interpretação visual ativa do jogador.

O BioLogic foi desenvolvido por alunos do Inteli e validado com especialistas da UNIFESP, com o objetivo de contribuir concretamente para a inclusão educacional e ampliar a autonomia de estudantes com deficiência no processo de aprendizagem.

#### 🔧 Como executar o código

Sem instalar nada, basta acessar o link abaixo no seu navegador:

<a href="https://graduacao.pages.git.inteli.edu.br/2026-1a/t27/g01">🎮 Acesse nosso jogo aqui!</a>

---

### Execução local — Passo a passo completo

Siga este guia se quiser rodar o jogo diretamente no seu computador. Não é necessário ter nenhuma experiência com programação.

---

#### Passo 1 — Instalar o Visual Studio Code (VS Code)

O VS Code é o programa que você usará para abrir e visualizar os arquivos do projeto.

1. Abra o seu navegador (Chrome, Edge, Firefox etc.) e acesse: **https://code.visualstudio.com**
2. Clique no botão azul grande **"Download for Windows"** (ou o equivalente para Mac/Linux, se for o seu caso).
3. Quando o download terminar, abra o arquivo baixado (ele terá um nome parecido com `VSCodeSetup-x64-....exe`).
4. Na janela de instalação que abrir:
   - Clique em **"Aceito o acordo"** e depois em **"Próximo"**.
   - Deixe as opções padrão e clique em **"Próximo"** nas telas seguintes.
   - Na tela de **"Tarefas adicionais"**, marque a opção **"Adicionar ao PATH"** (geralmente já vem marcada) e também marque **"Abrir com Code"** se aparecer.
   - Clique em **"Instalar"** e aguarde a instalação terminar.
   - Clique em **"Concluir"** para fechar o instalador.
5. Procure o ícone do VS Code na área de trabalho (um quadrado azul com um "V") e abra o programa para confirmar que funcionou.

---

#### Passo 2 — Instalar o Git

O Git é o programa que permite baixar ("clonar") o código do projeto do servidor para o seu computador.

1. No seu navegador, acesse: **https://git-scm.com/downloads**
2. Clique em **"Windows"** (ou no seu sistema operacional).
3. O download começará automaticamente. Quando terminar, abra o arquivo baixado (algo como `Git-2.xx.x-64-bit.exe`).
4. Na janela de instalação:
   - Clique em **"Next"** em todas as telas, mantendo as opções padrão.
   - Na tela **"Choosing the default editor used by Git"**, pode deixar como está e clicar em **"Next"**.
   - Continue clicando em **"Next"** até chegar em **"Install"**.
   - Clique em **"Install"** e aguarde.
   - Clique em **"Finish"** para fechar.
5. Para confirmar que o Git foi instalado, clique com o botão direito em qualquer lugar da área de trabalho. Se aparecer a opção **"Git Bash Here"** ou **"Open Git Bash here"** no menu, a instalação foi bem-sucedida.

---

#### Passo 3 — Instalar a extensão Live Server no VS Code

A extensão Live Server permite visualizar o jogo no navegador a partir do VS Code, como se fosse um servidor real.

1. Abra o **VS Code**.
2. No lado esquerdo, clique no ícone de quatro quadradinhos (parece um cubo quebrado) — esse é o menu de **Extensões**. Você também pode pressionar as teclas **Ctrl + Shift + X** ao mesmo tempo.
3. Na barra de pesquisa que aparecer no topo, digite: `Live Server`
4. O primeiro resultado deve ser **"Live Server"** de **Ritwick Dey**. Clique nele.
5. Clique no botão verde **"Instalar"** (Install).
6. Aguarde alguns segundos. Quando terminar, o botão mudará para "Desinstalar", o que confirma que a extensão foi instalada.

---

#### Passo 4 — Baixar o projeto (clonar o repositório)

Agora você vai baixar todos os arquivos do jogo para o seu computador.

1. Escolha uma pasta no seu computador onde quer salvar o projeto (por exemplo, dentro de **Documentos** ou na **Área de Trabalho**).
2. Abra essa pasta no **Explorador de Arquivos** do Windows.
3. Clique com o botão direito dentro da pasta (em um espaço vazio) e selecione **"Git Bash Here"** (ou "Open Git Bash here").
4. Uma janela preta com texto branco vai abrir — isso é o terminal do Git. Não se assuste, você só vai digitar um comando.
5. Digite o comando abaixo e pressione **Enter**:

```bash
git clone https://git.inteli.edu.br/graduacao/2026-1a/t27/g01.git
```

6. Aguarde o download terminar. Você verá mensagens aparecendo na tela. Quando o cursor piscar novamente em uma linha nova, o download terminou.
7. Uma nova pasta chamada `g01` terá sido criada no local que você escolheu.

---

#### Passo 5 — Abrir o projeto no VS Code

1. Abra o **VS Code**.
2. No menu superior, clique em **"File"** (Arquivo) e depois em **"Open Folder..."** (Abrir Pasta...).
3. Navegue até a pasta `g01` que foi criada no passo anterior e clique em **"Selecionar Pasta"**.
4. O VS Code vai carregar o projeto. No painel esquerdo, você verá todos os arquivos e pastas do jogo.

> **Dica:** Se aparecer uma janela perguntando "Do you trust the authors of the files in this folder?" (Você confia nos autores?), clique em **"Yes, I trust the authors"**.

---

#### Passo 6 — Rodar o jogo com o Live Server

1. No painel esquerdo do VS Code (o **Explorer**, que mostra os arquivos), clique na pasta **`public`** para expandi-la.
2. Dentro da pasta `public`, você verá um arquivo chamado **`index.html`**.
3. Clique com o botão direito sobre o arquivo **`index.html`**.
4. No menu que aparecer, clique em **"Open with Live Server"**.
5. O seu navegador padrão abrirá automaticamente com o jogo rodando.

> **Importante:** O jogo deve ser aberto a partir do arquivo `index.html` que está **dentro da pasta `public/`**. Se você abrir o `index.html` de outro lugar, as imagens e sons do jogo não vão carregar corretamente.

 Histórico de lançamentos

* Sprint 1 - 13/02/2026: Concepção do projeto e desenvolvimento preliminar. Definição da proposta do jogo, estruturação inicial do código e entrega da primeira tela funcional: menu com botão "Jogar" e navegação básica entre cenas.

* Sprint 2 - 27/02/2026: Desenvolvimento básico do jogo e game design. Criação da classe Button, implementação da primeira questão do quiz (bactéria como resposta implícita) e documentação do game design, visão geral do jogo, público-alvo, análise de mercado (5 Forças de Porter, SWOT) e missão/visão/valores.

* Sprint 3 - 13/03/2026: Desenvolvimento intermediário. Expansão do quiz para 10 questões jogáveis cobrindo citologia, evolução, genética e fotossíntese. Implementação do tutorial narrativo introdutório, criação da classe Helper para padronização da lógica de interação e melhorias visuais como efeito hover nos botões.

* Sprint 4 - 27/03/2026: Finalização do MVP e entendimento de negócio. Expansão para 15 questões, encerramento narrativo do tutorial e fechamento da história, sistema de feedback por questão (acerto e erro com explicação), tela final de desempenho do jogador. Documentação da proposta de valor, matriz de riscos e descrição da solução.

* Sprint 5 - 10/04/2026: Revisão e entrega final. Refinamentos gerais do jogo, documentação de testes de jogabilidade, revisão da documentação completa e validação dos critérios de publicação do projeto.
